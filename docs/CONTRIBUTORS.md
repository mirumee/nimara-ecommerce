## How to Become a Contributor

### 1. Get Started

- **Read the documentation** - Familiarize yourself with [Getting Started](./HOW_TO_USE.md)
- **Understand the architecture** - Review [Contributing Guide](./CONTRIBUTING.md)
- **Follow code style** - See [Code Style Guide](./CODE_STYLE_GUIDE.md)

### 2. Find Something to Work On

- **Check GitHub Issues** - Look for issues labeled `good first issue` or `help wanted`
- **Review the roadmap** - See [Vision & Roadmap](./VISION_AND_ROADMAP.md)
- **Ask questions** - Start a [Discussion](https://github.com/mirumee/nimara-ecommerce/discussions)

### 3. Submit Your Work

- **Fork the repository** - Create your own copy
- **Create a branch** - Use descriptive branch names
- **Make your changes** - Follow our conventions
- **Write tests** - Ensure quality
- **Open a Pull Request** - To the `develop` branch

## Types of Contributions

### Code Contributions

- **New features** - Implement planned features from roadmap
- **Bug fixes** - Fix reported issues
- **Performance improvements** - Optimize existing code
- **Refactoring** - Improve code quality
- **Tests** - Add or improve test coverage

### Documentation

- **Guides and tutorials** - Help others learn
- **API documentation** - Document code
- **Examples** - Show how to use features
- **Translations** - Translate docs to other languages

### Community

- **Answer questions** - Help others in Discussions
- **Triage issues** - Help organize and clarify issues
- **Review PRs** - Provide feedback on contributions
- **Share your work** - Blog posts, videos, talks

### Design

- **UI/UX improvements** - Enhance user interface
- **Design system** - Contribute to component library
- **Accessibility** - Improve accessibility
- **Branding** - Help with visual identity

## Community Guidelines

### Code of Conduct

We are committed to providing a welcoming and inclusive environment. All contributors must adhere to our [Code of Conduct](../CODE_OF_CONDUCT.md).

### Our Standards

**Positive behavior includes:**

- Using welcoming and inclusive language
- Being respectful of differing viewpoints
- Gracefully accepting constructive criticism
- Focusing on what is best for the community
- Showing empathy towards other community members

**Unacceptable behavior includes:**

- Harassment of any kind
- Trolling or insulting comments
- Personal or political attacks
- Publishing others' private information
- Other conduct inappropriate in a professional setting
