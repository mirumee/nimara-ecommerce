# Content

## Outcome

A storefront developer can configure required services, validate environment variables, run
Nimara locally, and reach a supported deployment target through one guided workflow.

## Scope boundary

The initiative improves the supported code-first path. It does not turn Nimara into a no-code
site builder and does not hide the composable services selected by the adopter.

## Evidence to collect

- Median time from clone to first successful local storefront.
- Completion rate of the guided setup without manual documentation searches.
- Number of invalid or missing environment values detected before build or deployment.
