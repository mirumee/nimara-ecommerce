## 2026-07-14

- **ADR**: Added [ADR-0002 Verified User Reviews on Saleor with nimara-mailer Invitations](tech/ADR/ADR-0002%20Verified%20User%20Reviews%20on%20Saleor%20with%20nimara-mailer%20Invitations.md) (Proposed) — a fresh, whole-system design for the [Verified User Reviews](product/epics/Epic%20User%20Reviews.md) epic, produced by a new `solution-author` grilling run (Session 2) at the user's request for a completely new ADR built from the epic, deliberately not anchored on ADR-0001. Decision: `reviews` as a swappable capability with the single `saleor` provider; each review a Saleor `Page` (`Review` PageType); per-product aggregate as product attributes (`review-rating-avg`/`review-count`, native filter/sort, churn accepted + rounding mitigation deferred); privileged writes via a storefront Server Action + Saleor app token after fulfilled-purchase verification; invitations owned by an **extended nimara-mailer** (`ORDER_FULFILLED` webhook → AWS-native multi-day delay → `review_invitation` React Email template, SES/SMTP). Weighed alternatives: dedicated Postgres, Product-metadata blob, metadata/hybrid aggregate, direct customer write, storefront cron + in-repo email adapter, and a second SaaS provider. No new repo dependency; no database. Appended Session 2 (`D-006`–`D-012`, `U-4`–`U-6`) to the [Verified User Reviews — Solution Grilling Log](product/solution/Verified%20User%20Reviews/Solution%20Grilling%20Log.md), registered ADR-0002 in the [ADR MOC](tech/ADR/ADR%20MOC.md), refreshed the grilling-log line in `index.md`, and linted the ADR clean (`pnpm wiki:adr:lint`, 0 errors / 0 warnings).
